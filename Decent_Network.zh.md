## Decent Network 低声网络

Decent Network 构建一个去中心化的好友网络（基于 DHT）：身份即公钥，通信以“成为好友”为许可，在线状态与事件天然存在；并在其上叠加存储、数据访问与智能能力，让 Web3 + AI 应用无需重复搭建隐形基础设施即可快速上线与扩展。

### 链之后的缺口
区块链擅长确权与可验证：资产、权限、凭据、状态。但一个应用要像现实世界一样“活着”，还需要链外的关键基础能力：  
- 即时通讯与实时互动（文字/语音/视频/会议）  
- 关系边界（谁能联系谁、谁能看到谁）  
- 离线内容与媒体（可用性、可访问性）  
- 数据访问（索引、缓存、聚合与查询）  
- 智能服务（推理、执行、工具调用）

AI 编程降低了开发门槛，但并不会自动解决这些系统性问题：可组合的权限边界、可扩展的互动与数据能力、以及避免平台锁定的迁移成本。

### 去中心化好友网络是“让协议活起来”的关键层
很多“链上身份/关系/社交协议”最终很难进入日常使用，一个重要原因是缺少真实互动层：身份与关系如果无法自然地转化为触达、互动与存在感，就很难产生持续留存与增长。

Decent Network 的底层是基于 DHT 的好友网络，围绕“连接与边界”建立默认规则：  
- 身份由用户自主生成（公钥即 ID），可携带，不依赖中心化账号体系  
- 双向通信以“成为好友”为许可；服务提供者同样遵循关系许可  
- 默认拒绝陌生人双向骚扰，把反垃圾与安全边界下沉为网络原语  
- 在线状态与事件是网络原生能力：在被允许的范围内传播“出现与活动”，形成更自然的社交与协作体验

这一层的意义在于：把网络效应沉淀到“身份/关系/互动”层，而不是沉淀在某个 App 的私有数据库里。

### 在其上叠加的服务能力
在好友网络的基础连接层之上，Decent Network 提供应用必需的可共享能力：  
- 存储：离线文件/媒体与内容可用性  
- 数据访问：查询、索引、缓存、聚合，让应用能顺畅地使用数据  
- 智能能力：模型/推理/执行能力的访问与组合，服务 AI 原生场景  
- 账号与资料：名称、资料、权限边界与共享策略

这让应用可以把精力用于“产品与增长”，而不是每次从零搭建整套隐形基础设施。

### Beagle Chat 的位置
Beagle Chat 是 Decent Network 的旗舰应用形态之一（iOS/Android/Web）：  
- 用真实用户验证底层网络与交互范式  
- 沉淀可复用的客户端能力（Beagle Chat Framework）  
- 为更多应用提供可参考的生产级实现

Decent Network 的目标不是“只做一个社交 App”，而是成为一条长期扩张的去中心化基础设施网络：更多节点提供能力，更多应用复用能力，更多用户带着身份与关系跨应用流动，最终形成可迁移、可复用、可持续的网络效应。
