### 重新定义社交网络的游戏规则：Decent Network 论文

在中心化社交平台肆意收割数据、操控信息流、审查内容并将用户锁定在企业云端的时代，**Decent Network** 提出了一种基础性转变：以设备为中心的点对点（P2P）协议，让社交回归真实生活——直接、无中介、用户掌控。

访问 https://decent.network 查看协议概览  
体验示例应用 https://beagle.chat（原生 Web3 即时通讯：P2P 端到端加密消息、音视频通话、离线消息、钱包身份、跨链资产、付费/代币门控会议、基于 Nostr 的 Moments，社区 16K+，下载 2K+）

#### 核心问题
Web2 社交依赖客户端-服务器云架构 → 隐私侵蚀、算法成瘾、中心化脆弱性，以及与自然人类互动的巨大偏差。

SocialFi（Farcaster、Lens 等）在现有链上叠加功能，却常常陷入过度金融化、激励错位（投机 > 实用）、用户体验差，以及 Vitalik Buterin 指出的**网络效应陷阱**：主流平台已形成自我强化，用户难以迁移。他呼吁从真正的社交原语出发，而不是在上面硬加加密。Suji Yan 则主张渐进式混合迁移，通过钱包/AI/治理整合逐步过渡。

解决方案不是更好的链上应用，而是为社交协调打造**基础协议层革命**，类似于区块链对金融的颠覆。

#### Decent Network：Friend Network 作为基础原语
Decent Network 使用 DHT（分布式哈希表）构建统一的 **Friend Network** —— 去中心化发现与路由，已在 BitTorrent 等系统中验证成熟、许可less、设备原生。

- **身份** = 公钥（用户自生成、可迁移；不依赖平台用户名）
- **关系** = 显式、双向好友，形成网络原生的访问控制列表（ACL）。陌生人默认无法发消息 → 内置反垃圾
- **P2P 连接** = 设备直连，支持多传输协议（WebRTC、QUIC 等）、NAT 穿越（ICE）、去中心化公钥目录。核心操作无需中心服务器
- **消息** = 端到端加密、基于权限的点对点（一对一或群组通过好友）
- **事件** = Friend Network 原生的短暂多播/广播原语：
  - 上线时，用户可广播可变的存在信息（可见性可配置）：在线/离线状态、显示名称、头像、自我介绍、可选位置/IP（仅限亲密好友以保护隐私）
  - 事件通过 gossip 协议传播给授权查看者（好友多播；可选广播给所有人/陌生人）
  - 用户随时更新
  - **关键：短暂性** —— 仅在上线时广播。离线 → 网络中无事件/存在感。完全模仿现实生活：你只在“出现”时被看见

这让网络“活”起来，支持实时可见性和协调，且无需持久云存储或中介。上层可叠加存储（离线文件）、数据访问（索引/查询）和智能（AI 推理/钩子）。

**Beagle Chat** 在生产环境中验证这些原语：P2P 消息/通话、钱包主权身份（SOL/EVM）、离线队列、代币激励等，全由 Friend Network 驱动。

#### 对 SocialFi 的意义
当前 SocialFi 是在金融水泥上盖房子。Decent Network 发明适合社交的“水泥”：设备优先的 P2P 基底，实现：
- 隐私默认（本地/加密数据，用户控制共享）
- 天然弹性（P2P 负载分担、无单点故障）
- 激励与实用对齐（可信中继微奖励，而非投机）
- 网络效应可复合：可迁移公钥身份 + 好友图跨应用流动

#### 攻克网络效应（最难的部分）
Vitalik 的头号障碍确实棘手。Decent 的应对：
- **开发者优先启动**：简单 TypeScript SDK → 嵌入钱包、DeFi、AI 代理或 Web2 应用，作为“特洛伊木马”
- **渐进/混合路径**：从小众起步（隐私/加密社区、受审查地区）；桥接到 X/Telegram；通过 QR/NFC 好友邀请，利用现实 proximity 病毒传播
- **实用飞轮**：实打实收益（拥有自己的图谱、无监控、直接通信）驱动有机增长，而非炒作
- **开放协作**：黑客松、资助、试点来教育和吸纳用户

#### 行动号召
Decent Network 不是又一个应用，而是 SocialFi 缺失的基础层：无许可、公钥寻址、好友绑定、P2P 通信，事件与消息直接在设备间流动。
