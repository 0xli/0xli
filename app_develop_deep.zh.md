## 1）核心命题

开发互动型 App 的难点不是“把页面做出来”，而是**以可控成本长期运营并持续扩展**。系统复杂度主要集中在通讯、触达、存储、身份与权限、数据访问与运维等基础能力上；重复造轮子会显著拖慢迭代速度与生态增长。

一个可持续演进的应用系统，需要在底层基础设施层面解决通用能力的复用问题，使上层产品能够专注于业务逻辑、用户体验与运营闭环。

---

## 2）为什么 AI 编程解决不了“系统问题”

AI 可以显著加速编码与原型开发，但以下关键能力仍然依赖清晰的系统架构、协议设计、数据模型与工程约束：

- 弱网环境与重连机制、并发控制与一致性、端侧状态机管理  
- Push 策略与到达率优化、通知系统与会话系统的联动  
- 离线文件与媒体管理、权限继承与访问控制机制  
- 账号、Profile、关系、订阅、会员等复杂权限模型  
- 数据查询、索引、审计与分析需求  
- 多端一致性与长期技术演进成本  

这些问题本质上是**系统工程问题**，而非单纯的代码生成问题。

## 3）Decent Network（底座网络）

Decent Network 是一个去中心化的消息、数据与智能服务网络，提供即时通讯、存储、账号体系与数据访问等应用必需能力，并支持对 Solana / Ethereum / Bitcoin 等公链以及模型推理能力的高效访问与调用。

在应用层面，它将“通讯 + 数据 + 智能”三类通用能力抽象为可复用的基础设施，使上层产品可以更加专注于业务闭环与运营。

在 Decent Network 中，运行在个人或组织手机、电脑等终端设备上的 **Agent 是构成去中心化朋友网络的重要且基础的节点**。这些 Agent 是客户端侧智能与算力的主要载体，既代表用户参与网络协作，也承担本地计算与执行能力。

它们不仅能够在 Web3 的朋友网络中进行高效、可信的通信与协作，还可以作为桥梁连接以平台智能与集中算力为主导的传统互联网世界，实现跨网络的资源调用与能力互通。通过这种双向连接，Decent Network 形成了兼具去中心化自治与外部可扩展性的混合计算与通信体系。

同时，Decent Network 原生支持 **AI Agent 作为系统的一等公民**。网络提供标准化的 Agent 通信与执行接口，包括 API / CLI / Webhook / Stream 等形式，使 Agent 可以像节点一样接入网络，进行消息交互、数据访问、链上操作与模型调用。系统支持 Agent 与人类、Agent 与 Agent 之间的高效协作与互操作，这是现代应用生态中不可或缺的基础能力。

## 4）Beagle Chat Framework（客户端框架）

Beagle Chat Framework 抽象自已上架应用商店的 Beagle Chat 客户端生产实现：

- 客户端结构：Swift / Kotlin + React Native  
- 标准交互模块：会话 / 消息 / 通知 / 媒体 / 权限  
- Web3 原生身份：钱包（Solana / Ethereum）+ 去中心化身份 + Name / Profile  

Beagle Chat 客户端在架构上将 **Agent 作为核心组成部分**。客户端不仅包含 Agent 的身份信息与运行环境，还负责管理与调度多种关键资源，包括：

- 本地设备算力与执行能力  
- 云端算力与智能服务的 API Key 与访问凭证  
- 加密货币资产及其钱包管理  
- 与身份、权限和数据访问相关的安全机制  

这些资源构成 Agent 在网络中参与生产与协作的基础生产资料。通过统一的客户端框架，用户与 Agent 可以安全、高效地管理和使用这些能力，在去中心化网络与外部智能服务之间建立稳定的运行环境。

Framework 在客户端层面同时提供 **Agent 集成能力**，支持将本地或远程 Agent 作为交互主体参与会话。通过统一的 API 与协议接口，应用可以轻松接入外部 Agent 服务，实现人机协作与多 Agent 协作的交互体验。

强调点：这不是概念性的设计框架，而是从真实运行的生产应用中沉淀出来的可复用实现与清晰的模型边界，同时兼顾人类用户与 Agent 用户的长期演进需求。

## 5）落地形态

- **独立 App 形态**：自有品牌 App 或网站接入 Framework 与 Network  
- **Mini App 形态**：以模块或小应用形式发布，并复用同一身份与网络能力  

不同形态的应用共享统一的底座能力与身份体系，降低开发成本并增强生态互联性。

---

## 6）设计理念（共生与可扩展）

身份与关键数据应具备更高的可迁移性与可组合性；应用共享底座基础设施与网络效应；网络资源通过按量计费、订阅、信用额度与 Token 等机制长期供给，形成可持续的生态增长结构。

在此基础上，系统采用 **Human + Agent 共生的设计理念**。Agent 被视为新的用户形态与计算节点，能够通过标准 API / CLI 接口参与网络协作。开放且可扩展的 Agent 生态促进应用之间的互联互通，使系统具备持续演进与规模化扩展的能力。

在这种架构下，人类用户与 Agent 共同构成网络的参与主体，形成兼具社交属性、计算能力与经济活动的统一生态系统。
